# Architecture Documentation

## Overview

DOCU-GEN follows Google's Software Engineering practices with:
- Clean OOP architecture
- Multi-provider LLM support
- Scalable test infrastructure
- Separation of concerns

## Documents

- **[Architecture Overview](./ARCHITECTURE.md)** - System architecture and OOP design
- **[LLM Providers](./LLM_PROVIDERS.md)** - Multi-provider support guide

## Key Design Principles

1. **Modularity**: Each component has a single responsibility
2. **Extensibility**: Easy to add new agents and providers
3. **Testability**: All components are testable in isolation
4. **Scalability**: Architecture supports growth
